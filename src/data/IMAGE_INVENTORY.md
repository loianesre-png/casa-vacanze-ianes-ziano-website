## Image Optimization Notes

- All apartment images are in **WebP** format ✅
- Good file sizes (200-1000 KB range)
- Already optimized for web delivery

### Existing Optimization Implementation

```typescript

```

This optimization strategy should be maintained in the new components.

## Recommendations for Phase 2-3

### 1. Keep Current Image References
The images are well-organized and the data structure in `apartments.ts` correctly references them. No renaming needed.

### 2. Image Component Pattern
When creating room showcase components, use the existing pattern:

```astro
const images = await Promise.all(
  imageModules.map(async (module) => {
    const optimized = await getImage({
      src: module.default,
      width: 1200,
      format: 'webp',
      quality: 85,
    });
    return optimized;
  })
);
```

### 3. Hero Images
For apartment page heroes, use:
- **Aspect ratio**: 16:9 or 21:9
- **Priority loading**: `loading="eager"` and `fetchpriority="high"`
- **Sizes**: Optimize for full-width display

### 4. Room Images
For room showcase sections:
- **Aspect ratio**: 4:3 or 16:9 (consistent within page)
- **Lazy loading**: `loading="lazy"` for below-fold images
- **Responsive**: Generate 3-4 size variants

### 5. Gallery/Carousel
The additional unused images (12 per Bilocale, 9 per Trilocale, 7 per Suite Deluxe) can be:
- Added to ImageCarousel components
- Used for room detail galleries
- Featured in "virtual tour" sections

## Next Steps for Phase 2

When creating apartment page components:

1. **Import images from data structure**:
   ```typescript
   import { getApartmentBySlug } from '~/data/apartments';
   const apartment = getApartmentBySlug('bilocale');
   ```

2. **Load images dynamically**:
   ```typescript
   const heroImage = await import(`../../assets/images/${apartment.id}/${apartment.heroImage}`);
   const roomImages = await Promise.all(
     apartment.rooms.map(room =>
       import(`../../assets/images/${apartment.id}/${room.image}`)
     )
   );
   ```
